# Contributing

This repository is proprietary and maintained by Zack Sikkink.

External contributions are not accepted by default.

If you need to propose a change:

1. Open an issue describing the problem and proposed approach.
2. Wait for written approval before preparing a patch.
3. Keep changes scoped and production-safe.
4. Include tests and documentation updates for behavioral changes.

By submitting any patch, you confirm you have permission to contribute that work.
